## 2024-03-24 - Accessibility patterns in icon-only buttons\n**Learning:** Custom chat interfaces often rely heavily on icon-only buttons for a clean aesthetic (like Paperclip, Mic, or Link icons), but developers frequently forget to add screen reader support (aria-labels) or hover context (titles) since the visual meaning seems obvious to sighted users.\n**Action:** Always audit interactive icon components in chat interfaces for `aria-label` and `title` attributes. Adding these micro-UX improvements provides essential context for screen readers and helpful tooltips for all users.
