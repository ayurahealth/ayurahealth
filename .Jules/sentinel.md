## 2025-02-23 - Removed CEO Authentication Bypass Backdoor
**Vulnerability:** A hidden backdoor (`CEO_BYPASS_KEY`) was present in `app/api/auth/ceo-pass/route.ts` and `app/api/chat/route.ts`. It allowed unauthenticated users to bypass Clerk authentication, rate limiting, and payment limits if they knew the secret key and set a specific browser cookie (`ayura_ceo_token`).
**Learning:** Custom bypass mechanisms create significant security risks. If the secret is exposed or weak, it grants full unmetered access to AI credits and system functionality, acting as a complete authentication bypass. This is a critical security vulnerability that circumvents standard auth and rate-limiting logic.
**Prevention:** Do not implement custom authentication backdoors or "God mode" endpoints based on shared secrets. Administrative or privileged access must be managed securely through established identity providers (like Clerk) using role-based access control (RBAC) via user metadata (e.g., `publicMetadata.role`).
