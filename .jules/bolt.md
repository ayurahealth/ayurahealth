## 2025-02-18 - Next.js App Router Memoization for List Items
**Learning:** In Next.js App Router client components, if expensive UI components like `ClinicalMarkdown` are wrapped in a list and rendered via `map()`, they will re-render fully during parent component updates (like streaming responses or keystrokes). If event handlers passed as props (`handleSpeak`, `onSelectSource`) are not wrapped in `useCallback` or are passed as inline functions `() => {}`, they undergo referential changes on every parent render, defeating standard React diffing and rendering optimizations.
**Action:** Always extract and memoize list event handlers with `useCallback` in the parent, and wrap the list item component with `React.memo` to prevent costly full-list DOM reconciliation and redundant CPU processing.
