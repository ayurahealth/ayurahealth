## 2024-04-23 - Heavy List Item Memoization in Next.js App Router
**Learning:** In Next.js App Router, rendering heavy components (like Markdown renderers inside chat messages) within a client component that manages real-time input state causes severe input lag due to unnecessary re-renders of the entire list on every keystroke. Wrapping list items with `React.memo` isn't enough; all callback props passed from the parent (e.g. event handlers like `onSpeakText` and `onSelectSource`) must also be memoized with `useCallback` to ensure stable references and actually prevent the re-render.
**Action:** When implementing lists of heavy components (especially those rendering Markdown) in components that also manage frequent state updates (like typing in a text input), always wrap the list items in `React.memo` and memoize all callback props with `useCallback`.
