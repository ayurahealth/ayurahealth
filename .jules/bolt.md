## 2025-02-12 - Prevent full-list re-renders for streaming components
**Learning:** In Next.js App Router client components, if a list of items is continuously updated (e.g., during streaming where the last item frequently changes or new items are added), mapping over heavy components like `MessageItem` (which renders Markdown) without `React.memo` causes the entire list to re-render on every keystroke or streaming update. Passing unmemoized inline functions (e.g., `() => {}`) or unmemoized handlers down to the item prop breaks `React.memo` entirely, rendering it useless.
**Action:** When working with dynamically updated lists, ensure list item components are wrapped in `React.memo` and that all function props (like event handlers) passed to them are stable using `useCallback` with exact dependency arrays.
