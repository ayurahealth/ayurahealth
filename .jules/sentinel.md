## 2024-05-13 - Fix Timing Attack Vulnerability in CEO Bypass Key Verification
**Vulnerability:** The `CEO_BYPASS_KEY` was being compared to user-provided input (query param `key` or cookie `ayura_ceo_token`) using the standard string equality operator `===`. This exposes the endpoint to timing attacks, allowing a potential attacker to deduce the bypass key character-by-character based on the time it takes the server to respond.
**Learning:** Hardcoded cryptographic or sensitive tokens must never be compared using simple string equality. Although the feature is meant to bypass standard authentication, it still requires secure cryptographic token handling to prevent unauthorized access.
**Prevention:** Always use `crypto.timingSafeEqual` with `Buffer.from(String(input))` to ensure constant-time token verification when comparing secure tokens, bypass keys, secrets, or passwords. Ensure buffer lengths are verified before comparison to avoid runtime errors that could lead to 500 responses.
