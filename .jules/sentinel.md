## 2025-02-14 - Timing Attack Vulnerability in Secret Comparisons
**Vulnerability:** Several API endpoints (`app/api/webhooks/razorpay/route.ts`, `app/api/razorpay/create-order/route.ts`, `app/api/auth/ceo-pass/route.ts`, `app/api/chat/route.ts`) used the standard equality operator (`!==` or `===`) to compare sensitive secrets like webhook signatures and bypass keys.
**Learning:** Using standard equality operators for string comparison can leak information about the secret length and characters due to early-exit optimizations, enabling a timing attack where an attacker guesses the secret byte-by-byte. This existed because the built-in string comparison in JavaScript is not constant-time.
**Prevention:** Always use `crypto.timingSafeEqual` with `Buffer.from()` when comparing sensitive tokens, secrets, or passwords. Check `buffer.length === expected.length` before calling `timingSafeEqual` to avoid errors, and ensure values passed to `Buffer.from()` are explicitly cast to strings or checked for truthiness. In Next.js, consider dynamic imports (`await import('crypto')`) to avoid edge module resolution issues.
