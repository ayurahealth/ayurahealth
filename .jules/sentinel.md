## 2025-05-24 - Timing Attacks in Authentication and Signature Checking
**Vulnerability:** Found timing attack vulnerabilities due to the use of strict equality operators (`===` and `!==`) to compare sensitive tokens and signatures. Affected files include Razorpay webhook signature validation (`app/api/webhooks/razorpay/route.ts`) and CEO bypass token validation (`app/api/auth/ceo-pass/route.ts`, `app/api/chat/route.ts`).
**Learning:** These vulnerabilities existed because developers used standard string equality operators rather than utilizing constant-time comparison methods. This allows attackers to potentially guess signatures or tokens byte-by-byte by analyzing response time variations.
**Prevention:** Always use `crypto.timingSafeEqual` with `Buffer.from` instead of standard equality operators (`===`) when comparing sensitive tokens, secrets, or passwords. Ensure buffer lengths are explicitly matched before calling `crypto.timingSafeEqual` to prevent Node from throwing an error.
