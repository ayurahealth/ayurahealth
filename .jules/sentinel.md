## 2026-04-25 - Timing Attacks on API Secrets
**Vulnerability:** The `CEO_BYPASS_KEY` authentication checks in `app/api/auth/ceo-pass/route.ts` and `app/api/chat/route.ts` were using strict equality (`===` and `!==`) to compare the user-provided token with the secret key. This exposes the API to timing attacks where an attacker could deduce the key character by character based on the time it takes the server to respond.
**Learning:** Vercel edge/Node.js environments require careful handling when importing built-in modules like `crypto`. Furthermore, `Buffer.from()` requires a truthy value (or explicit casting/fallback to an empty string) and throws an error if undefined or null is passed. `crypto.timingSafeEqual` also throws an error if the two buffers are not the exact same length. All three of these edge cases must be addressed.
**Prevention:** Always use `crypto.timingSafeEqual` to compare sensitive tokens or passwords. First check if both exist, then convert them to buffers using `Buffer.from(token)`. Verify `bufferA.length === bufferB.length` before calling `timingSafeEqual`. If inside a Next.js API route that may execute on the edge, consider dynamically importing `crypto` via `const crypto = await import('crypto')`.
